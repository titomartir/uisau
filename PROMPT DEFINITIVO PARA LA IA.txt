PROMPT DEFINITIVO PARA LA IA
Rol que debe adoptar la IA:
Actúa como un experimentado Arquitecto de Software e Ingeniero Full-Stack. Tu tarea es generar el código base completo, estructurado y funcional, para una aplicación web de gestión y aplicación de encuestas de satisfacción. El código debe seguir las mejores prácticas, ser escalable y estar listo para ejecutarse en un entorno de desarrollo con Windows 11, VS Code, Node.js, React y PostgreSQL.

Contexto del Proyecto:
Necesito una aplicación para realizar múltiples encuestas de satisfacción del cliente. Como base, tengo una encuesta modelo en formato PDF ("Encuesta de Satisfacción del Usuario en la Red de Hospitales Nacionales"). La aplicación debe permitir, en el futuro, cargar diferentes cuestionarios, pero inicialmente debe implementar fielmente la estructura y preguntas de este PDF.

DECISIONES TOMADAS (Basadas en las 5 mejoras sugeridas):
Autenticación del Panel Admin:

Usaré JWT (JSON Web Tokens) con refresh token para mayor robustez.

Credenciales iniciales: admin@example.com / Admin123! (configurable en .env)

Estructura preparada para futura integración con roles de usuarios.

Gestión de Respuestas en Admin:

Funcionalidades: Ver (con paginación, búsqueda y filtros) y Exportar a CSV/Excel.

NO incluir edición o borrado de respuestas en versión inicial (preservar integridad de datos).

Botón de "marcar como revisada" para seguimiento administrativo.

Campo "Servicio donde fue atendido":

Lista desplegable de selección única con opciones: "Consulta Externa", "Emergencia", "Encamamiento".

Si selecciona "Encamamiento", se activa la sección correspondiente (preguntas sobre ropa de cama y alimentos).

Sección "Servicios de Apoyo":

Mostrar checkboxes para que el usuario seleccione qué servicios recibió.

Dinámicamente, mostrar las preguntas de satisfacción SOLO para los servicios seleccionados.

Si no selecciona ninguno, omitir toda la sección.

Estrategia de Entrega de Código:

Primera respuesta: Entregar estructura completa del proyecto + backend completo (models, controllers, routes, seeders, middleware de autenticación).

Segunda respuesta: Frontend completo (componentes, páginas, contexto/estado, servicios API, lógica condicional).

Tercera respuesta: README.md detallado con instrucciones de instalación, configuración, y ejemplos de uso.

Base de Datos (PostgreSQL):
Nombre de la DB: encuestas_satisfaccion

Modelos detallados (incluyendo restricciones y relaciones):

Tabla usuarios (para admin):

id (PK), email (unique), password_hash, rol, created_at, last_login

Tabla encuestas:

id (PK), nombre, descripcion, version, activa (boolean), fecha_creacion

Tabla preguntas:

id (PK), encuesta_id (FK), texto_pregunta, tipo_respuesta (enum: 'likert_5', 'texto', 'si_no', 'seleccion_unica', 'fecha', 'hora'), categoria, orden, dependencia (JSONB), requerido (boolean)

Tabla opciones_respuesta:

id (PK), pregunta_id (FK), valor_texto, puntaje (int, para likert), orden

Tabla respuestas_encabezado:

id (PK), encuesta_id (FK), fecha_inicio TIMESTAMP, fecha_fin TIMESTAMP,

Campos demográficos: origen_etnico, edad (int), sexo, departamento, municipio, hospital, servicio (enum: 'consulta_externa', 'emergencia', 'encamamiento')

Contacto: telefono (encriptado), email, acepta_contacto (boolean)

Metadata: ip_address, user_agent, created_at

Tabla respuestas_detalle:

id (PK), respuesta_encabezado_id (FK), pregunta_id (FK), opcion_id (FK, nullable), respuesta_texto (TEXT), created_at
